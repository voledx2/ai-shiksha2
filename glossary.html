AI Glossary

Every term you'll encounter in LLMs, RAG, agents, and alignment — explained with precision and the context missing from Wikipedia.

🔍
A
Attention Mechanism Architecture

A function that computes a weighted sum of values (V) based on the compatibility between queries (Q) and keys (K). In transformers, attention allows each token to "attend to" every other token in the sequence. Scaled dot-product attention computes scores as QKᵀ/√d_k, applies softmax, then multiplies by V.

💡 The division by √d_k prevents the dot products from growing large in high dimensions, which would push softmax into saturation and cause vanishing gradients.
→ See: Attention Mechanisms course
Autoregressive Model Architecture

A model that generates sequences one token at a time, with each new token conditioned on all previously generated tokens. GPT-style LLMs are autoregressive — they predict P(xₜ | x₁, x₂, ..., xₜ₋₁) at each step using a causal (masked) attention mechanism.

Alignment Safety

The problem of ensuring that AI systems behave in ways that reflect human intentions and values. Technical alignment approaches include RLHF, DPO, Constitutional AI, and interpretability research. Distinguished from "capability" — a highly capable but misaligned model can be dangerous.

B
BPE (Byte Pair Encoding) Architecture

A subword tokenization algorithm that iteratively merges the most frequent byte pairs in a corpus. Starting from individual characters (or bytes), BPE builds a vocabulary of variable-length subwords. Most modern LLMs (GPT, LLaMA, Mistral) use BPE or a variant.

💡 BPE balances vocabulary size against sequence length. A smaller vocabulary means longer sequences; a larger vocabulary handles rare words better but increases embedding table size.
BM25 RAG

Best Match 25 — a classical sparse retrieval algorithm based on TF-IDF with length normalization. BM25 scores documents by term frequency, inverse document frequency, and document length penalty. Despite its simplicity, it remains competitive in hybrid retrieval setups when combined with dense vector search.

C
Constitutional AI (CAI) Safety

Anthropic's method of training AI systems to be harmless using AI feedback (RLAIF) rather than human feedback. A set of principles (the "constitution") guides the AI to critique and revise its own outputs, replacing human labelers in the RLHF loop for harmlessness training.

→ See: Constitutional AI paper annotation
Chunking RAG

The process of splitting documents into smaller segments for storage in a vector database. Chunking strategy significantly affects RAG quality. Fixed-size chunking splits by token count; semantic chunking splits on meaning boundaries; recursive chunking uses hierarchy (sections → paragraphs → sentences).

Context Window Architecture

The maximum number of tokens an LLM can process in a single forward pass — both input (prompt) and output (completion) combined. Modern models range from 4k (early GPT-4) to 1M+ tokens (Gemini 1.5 Pro). Longer contexts require techniques like GQA, sliding window attention, and specialized positional encodings.

D
DPO (Direct Preference Optimization) Training

A fine-tuning algorithm that aligns LLMs to human preferences without training a separate reward model. DPO reformulates the RLHF objective to directly optimize the policy using preference pairs (chosen vs rejected responses), showing that the language model itself is implicitly a reward model.

→ See: DPO paper annotation
Dense Retrieval RAG

Retrieval using dense vector representations (embeddings) rather than sparse keyword matching. A bi-encoder encodes queries and passages separately; retrieval finds nearest neighbors in embedding space using approximate nearest neighbor (ANN) search. Contrasted with sparse retrieval (BM25) and late-interaction (ColBERT).

E
Embedding Architecture

A dense vector representation of text (token, sentence, or document) in a continuous high-dimensional space. Semantically similar texts have embeddings that are close in this space (measured by cosine similarity or dot product). Embeddings are the fundamental representation used in both transformers and retrieval systems.

F
Fine-Tuning Training

Updating a pre-trained model's weights on a smaller, task-specific dataset. Supervised Fine-Tuning (SFT) uses labeled examples; RLHF fine-tunes with human preference data. Full fine-tuning updates all parameters; parameter-efficient methods (LoRA, adapters) update only a small subset.

Function Calling Agents

A capability where LLMs can generate structured output (JSON) describing a tool to call with specific arguments, rather than generating prose. The application executes the function and passes results back to the model. The foundation of tool-using agents and the MCP protocol.

G
GQA (Grouped-Query Attention) Architecture

An attention variant where multiple query heads share a single key-value head. Reduces KV cache memory by a factor equal to the number of query groups, enabling larger batch sizes or longer contexts without proportional memory cost. Used in LLaMA 2 70B, Mistral, and most modern large models.

H
HNSW (Hierarchical Navigable Small World) RAG

A graph-based approximate nearest neighbor (ANN) index used in vector databases. HNSW builds a layered graph where each node connects to its nearest neighbors at multiple scales. Offers excellent recall-speed trade-offs and is used by Pinecone, Weaviate, Qdrant, and most production vector stores.

K
KV Cache Architecture

During autoregressive inference, keys and values for all previous tokens are cached to avoid recomputation at each generation step. Without KV caching, inference time is O(n²) in sequence length; with caching, it's O(n) per step. KV cache size is the primary memory bottleneck in production serving.

KL Divergence Training

Kullback-Leibler divergence — a measure of how one probability distribution differs from another. In RLHF, a KL penalty term constrains the fine-tuned model from deviating too far from the reference (pre-trained) model, preventing reward hacking and maintaining language fluency.

L
LoRA (Low-Rank Adaptation) Training

A parameter-efficient fine-tuning method that inserts trainable low-rank matrices (ΔW = AB) into pre-trained weight matrices while keeping original weights frozen. If a matrix W ∈ ℝᵐˣⁿ, LoRA approximates the update with A ∈ ℝᵐˣʳ and B ∈ ℝʳˣⁿ where r ≪ min(m,n). Dramatically reduces trainable parameters (typically 0.1–1% of total).

LLM (Large Language Model) Architecture

A neural language model with billions of parameters, pre-trained on large text corpora using next-token prediction. Modern LLMs are transformer-based and exhibit emergent capabilities (instruction following, in-context learning, reasoning) not explicitly trained for.

M
MCP (Model Context Protocol) Agents

Anthropic's open protocol for connecting LLMs to external tools, data sources, and services. MCP defines a standardized client-server architecture where MCP servers expose tools (functions), resources (data), and prompt templates, while MCP clients (LLM applications) query them at runtime.

→ See: MCP Specification paper annotation
Mixture of Experts (MoE) Architecture

An architecture where each transformer layer contains multiple "expert" feed-forward networks, and a learned router selects a sparse subset (typically top-2) of experts for each token. MoE scales model parameters without proportionally increasing FLOP cost — Mixtral 8x7B activates ~13B parameters per token despite having ~47B total.

Multi-Head Attention Architecture

Running h parallel attention operations ("heads") with separate Q, K, V projection matrices, then concatenating and projecting outputs. Different heads can specialize in different types of relationships (syntactic, semantic, positional). The key operation in every transformer layer.

P
Perplexity Architecture

Prompt Engineering Architecture

The practice of crafting input prompts to elicit desired behavior from an LLM without updating model weights. Techniques include zero-shot prompting, few-shot exemplars, chain-of-thought reasoning, role assignments, and structured output specification. Effective prompting can dramatically change model outputs.

Q
Quantization Architecture

Reducing the numerical precision of model weights (and sometimes activations) to decrease memory footprint and increase inference speed. Common formats: FP16 (16-bit), INT8 (8-bit), INT4 (4-bit). Methods include GPTQ (post-training quantization), AWQ (activation-aware), and GGUF (used by llama.cpp for CPU inference).

R
RAG (Retrieval-Augmented Generation) RAG

A pattern that augments LLM generation with relevant documents retrieved from an external knowledge base. At inference time, a retriever fetches top-k relevant chunks for the query; these are injected into the prompt as context before generation. RAG reduces hallucinations on knowledge-intensive tasks and allows knowledge updates without retraining.

→ See: RAG Track courses
RLHF (Reinforcement Learning from Human Feedback) Training

A training paradigm that fine-tunes LLMs using human preference data. Three steps: (1) supervised fine-tuning on demonstrations, (2) training a reward model on human preference pairs, (3) RL fine-tuning (typically PPO) to maximize reward while constraining KL divergence from the SFT model. Used in InstructGPT, GPT-4, Claude, and most instruction-following models.

RoPE (Rotary Position Embedding) Architecture

A positional encoding scheme that encodes position by rotating query and key vectors in 2D subspaces using complex multiplication. RoPE naturally encodes relative positions (Qₘ · Kₙ depends only on m-n), enables length extrapolation beyond training length, and is used in LLaMA, Mistral, Gemma, and most modern open-source LLMs.

S
Softmax Architecture

A function that converts a vector of real-valued scores into a probability distribution: softmax(xᵢ) = exp(xᵢ) / Σⱼ exp(xⱼ). Used in attention (to normalize scores into weights) and in the final layer (to produce next-token probabilities). Temperature scaling divides logits before softmax to control distribution sharpness.

Speculative Decoding Architecture

A technique to accelerate LLM inference where a small draft model generates k candidate tokens, then the large target model verifies all k tokens in a single parallel forward pass. Accepted tokens are kept; the first rejected token is corrected. Can achieve 2-3x speedup with no quality loss when draft and target are well-matched.

Superposition Safety

The phenomenon where neural networks represent more features than they have neurons by encoding multiple features in the same neurons (polysemanticity). Features are represented as nearly-orthogonal directions in activation space. Understanding superposition is central to mechanistic interpretability research.

T
Transformer Architecture

The dominant neural network architecture for NLP, introduced in "Attention Is All You Need" (Vaswani et al., 2017). Key components: multi-head self-attention, position-wise feed-forward networks, residual connections, and layer normalization. Replaces RNNs by enabling parallel computation over sequences. All modern LLMs are transformer-based.

Tokenization Architecture

The process of converting raw text into discrete tokens (integers) that an LLM can process. Tokens represent subword units rather than characters or words — "tokenization" might become ["token", "ization"] depending on the vocabulary. Tokenizer design affects model performance, multilingual capability, and efficiency.

V
Vector Database RAG

A database optimized for storing and querying high-dimensional embedding vectors using approximate nearest neighbor (ANN) search. Popular options: Pinecone (managed), Weaviate (open-source), Qdrant (open-source), pgvector (PostgreSQL extension), Chroma (lightweight). Core infrastructure component in RAG systems.
